## 背景与问题

动态算力分配需要分层触发，记忆需要多层沉淀与检索。

---

## 核心方案：多层记忆架构

### 记忆层级

| 层级 | 名称 | 存储介质 | 生命周期 | 类比 |
|------|------|----------|----------|------|
| L1 | Working Memory | 内存 | 当前 API 请求 | CPU 寄存器 |
| L2 | State Memory | 文件 (PLAN.md/TODO.md) | 任务周期 | L2 缓存 |
| L3 | Episodic Memory | 文件 (YYYY-MM-DD.md) | 跨会话 | L3 磁盘 |
| L4 | Long-term Memory | MEMORY.md + Vector DB | 永久 | 冷备存储 |

### 动态算力分配触发条件

**宏观触发**：
- 外部记忆文件（PLAN.md）检测到任务目标发生变更
- 下一 Turn 开始前开启慢思考，重新校准执行策略

**微观触发**：
- 某工具调用返回非预期结果（Error 或严重偏差输出）
- 当前 Turn 内动态开启慢思考，对局部异常推理后再决定下一步

### Agent Memory 的多层沉淀

**工作区目录结构**：
```
~/.openclaw/workspace/
├── memory/
│   ├── 2026-04-12.md      # 按日期自动生成
│   ├── 2026-04-13.md
│   └── ...
├── MEMORY.md              # 长期事实、偏好、关键决策汇总
└── plans/
    ├── PLAN.md            # 当前任务计划
    └── TODO.md            # 当前任务清单
```

**Episodic Memory 沉淀时机**：
- 上下文即将被 Compaction 之前
- 自动触发静默轮次，将未落盘的重要信息写入记忆文件

**Dreaming 机制（实验性）**：
- 后台对短期记忆信号进行评分
- 高质量条目自动晋升至长期记忆 MEMORY.md

### 长程记忆检索（Hybrid Retrieval）

当用户提出跨越周期的历史性问题时，Agent 自动调用 `memory_search` 工具：

**混合检索策略**：
1. **向量搜索**：语义相似性，理解近义表达
2. **BM25 关键词搜索**：精确匹配 ID、错误字符串、配置键名
3. **合并排序**：两路结果综合评分

**Embedding 提供商**：
- OpenAI、Gemini、Voyage、Mistral（开箱即用）
- 本地模型（完全离线语义检索）
