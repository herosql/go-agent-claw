### 真正的沙箱隔离（Sandboxing）

在面向多租户的云端场景下，单纯的 Middleware 拦截是不够的。你需要探索如何利用以下技术，为每个 Agent 分配一个用完即毁、绝对隔离的物理执行沙箱：

- **Docker**：容器化隔离，每个 Agent 任务运行在独立容器中
- **Firecracker（MicroVM）**：轻量级 MicroVM，比容器更安全的隔移
- **WebAssembly（Wasm）**：语言级别的沙箱隔离，适合插件场景

**待实现**：
- [ ] Docker 沙箱管理器
- [ ] Firecracker MicroVM 集成
- [ ] Wasm 插件执行环境
- [ ] 沙箱资源限制（CPU、内存、磁盘、网络）

### 企业级的细粒度权限体系（Permission System）

就像 Claude Code 广受好评的 `settings.json` 设计一样，你需要将"高危拦截"从硬编码中剥离出来，构建一套灵活的策略引擎。

让终端用户可以自定义某个 Bash 命令到底是处于：
- **allow**：无脑放行
- **ask**：弹窗审批
- **deny**：绝对禁止

的权限级别。

**待实现**：
- [ ] 策略引擎数据结构设计
- [ ] `settings.json` 风格的权限配置解析
- [ ] 高危命令自动识别与分类
- [ ] 审批流程与飞书集成

### 智能体自进化（Agentic Self-Evolution）

这是目前学术界和工业界都在疯狂探索的圣杯（工业界如 Hermes 项目便是一个典型代表）。

在这个阶段，Harness 不仅仅是"跑"大模型，而是能够在"闲时"去收集和扫描每次执行失败的 Trace 链路和人类的纠偏记录，像"人类做梦"一样提取出高频的报错规律（比如"为什么每次重构都会遗漏路由注册"）。

然后，它会自动去修改并更新工作区内的 AGENTS.md 以及相关 skills，把这些沉淀下来的教训变成新的标准作业程序（SOP）。

这样，引擎在日复一日的运行中，就能实现真正脱离人类干预的"经验自我迭代"。

**待实现**：
- [ ] 失败 Trace 自动收集与分类
- [ ] 人类纠偏记录解析
- [ ] 高频错误模式挖掘算法
- [ ] AGENTS.md 自动更新机制
- [ ] SOP 生成与沉淀逻辑
